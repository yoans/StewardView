# StewardView Security Documentation

This directory contains security policies and compliance documentation for StewardView's production operations.

## Documents

| Document | Purpose |
|---|---|
| [Information Security Policy](information-security-policy.md) | Master security policy covering all controls |
| [Incident Response Plan](incident-response-plan.md) | Detection, containment, notification, and remediation procedures |
| [Data Retention and Deletion Policy](data-retention-policy.md) | What data is kept, for how long, and how it is deleted |
| [Acceptable Use Policy](acceptable-use-policy.md) | BYOD, credential handling, and system access rules |
| [Security Awareness Training Policy](security-awareness-training.md) | Onboarding and annual training requirements |
| [Vendor Risk Management Policy](vendor-risk-management.md) | Vendor intake, monitoring, and offboarding |

## Document Maintenance

Fill in the following placeholders throughout the documents:

- **[Platform Administrator Name]** — the name of the person responsible for security
- **[security contact email]** — the email used for security communications
- **[Name] — [email] — [phone]** in the Incident Response Plan contact table
